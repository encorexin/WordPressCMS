## 第一阶段：基础体验优化

**目标**：提升基础使用体验，增强系统稳定性，建立良好的开发基础。

**预计工期**：3-5 天

### 任务清单

#### 1.1 全局搜索功能

**需求描述**：
- 支持 `Cmd/Ctrl + K` 快捷键唤起搜索
- 可搜索文章、关键词、模板、主题
- 支持模糊匹配，实时显示结果
- 点击结果直接跳转

**实现步骤**：
1. 创建 `CommandDialog` 组件（基于 shadcn Command）
2. 实现搜索逻辑，从 IndexedDB 检索数据
3. 添加键盘快捷键监听
4. 结果分类展示（文章/关键词/模板/主题）

**验收标准**：
- [ ] 快捷键可正常唤起/关闭搜索
- [ ] 输入关键词实时显示匹配结果
- [ ] 点击结果正确跳转到对应页面
- [ ] 空状态提示友好

---

#### 1.2 快捷键支持

**需求描述**：
- 文章编辑页：`Ctrl+S` 保存、`Ctrl+P` 发布
- 全局：`Ctrl+K` 打开搜索、`?` 显示快捷键帮助
- 提供快捷键帮助面板

**实现步骤**：
1. 创建 `useHotkeys` Hook 统一管理快捷键
2. 在 ArticleEditor 页面绑定编辑相关快捷键
3. 在 Header 添加帮助按钮和快捷键面板
4. 防止快捷键冲突（如输入法状态）

**验收标准**：
- [ ] `Ctrl+S` 保存当前文章
- [ ] `Ctrl+P` 发布文章（需确认弹窗）
- [ ] `Ctrl+K` 打开全局搜索
- [ ] `?` 显示快捷键帮助面板
- [ ] 快捷键面板展示所有可用快捷键

---

#### 1.3 错误边界（Error Boundary）

**需求描述**：
- 捕获组件渲染错误，防止白屏
- 提供友好的错误提示页面
- 支持刷新恢复或返回首页

**实现步骤**：
1. 创建 `ErrorBoundary` 类组件
2. 设计错误 fallback UI
3. 在 App.tsx 包裹主要路由
4. 添加错误日志记录

**验收标准**：
- [ ] 组件报错时显示错误页面而非白屏
- [ ] 错误页面包含：错误信息、刷新按钮、返回首页按钮
- [ ] 错误信息可折叠查看详情
- [ ] 支持手动恢复尝试

---

#### 1.4 页面加载优化

**需求描述**：
- 路由级组件懒加载
- 添加页面加载骨架屏

**实现步骤**：
1. 使用 `React.lazy` 改造路由组件
2. 创建 `PageSkeleton` 加载占位组件
3. 配置路由 Suspense

**验收标准**：
- [ ] 首屏加载 JS 体积减少
- [ ] 页面切换显示骨架屏
- [ ] 无明显的加载卡顿

---

### 第一阶段完成检查项

- [x] 全局搜索功能完整可用
- [x] 快捷键系统正常工作
- [x] 错误边界捕获测试通过

## 第二阶段：核心功能增强

**目标**：增强核心内容管理功能，提升数据安全性和操作效率。

**预计工期**：5-7 天

### 任务清单

#### 2.1 文章版本历史

**需求描述**：
- 自动保存文章编辑历史（每 30 秒或手动触发）
- 支持查看历史版本列表
- 支持版本对比（diff）
- 支持回滚到指定版本

**实现步骤**：
1. 扩展数据库表 `article_versions`
2. 在 ArticleEditor 添加自动保存逻辑
3. 创建版本列表页面/弹窗
4. 集成 diff 库（如 diff-match-patch）
5. 实现版本回滚功能

**验收标准**：
- [ ] 编辑文章时自动保存版本
- [ ] 可查看文章的所有历史版本
- [ ] 版本对比清晰显示增删改
- [ ] 回滚功能正常工作
- [ ] 最多保留 50 个版本，自动清理旧版本

---

#### 2.2 批量操作功能

**需求描述**：
- 文章列表支持多选
- 批量发布、批量删除、批量移动站点
- 批量操作前确认弹窗
- 显示批量操作进度

**实现步骤**：
1. 在 Articles 页面添加多选功能
2. 实现批量操作工具栏
3. 创建批量操作确认弹窗
4. 实现批量操作逻辑（串行处理避免请求过载）
5. 添加进度提示

**验收标准**：
- [x] 可多选文章（支持全选/取消全选）
- [ ] 批量发布功能正常
- [x] 批量删除有二次确认
- [x] 操作过程显示进度
- [ ] 操作完成后显示成功/失败统计

---

#### 2.3 虚拟滚动优化

**需求描述**：
- 文章列表大数据量流畅滚动
- 关键词列表虚拟化
- 保持滚动位置记忆

**实现步骤**：
1. 集成 `react-window` 或 `@tanstack/react-virtual`
2. 改造 Articles 列表组件
3. 改造 Keywords 列表组件
4. 添加滚动位置恢复

**验收标准**：
- [ ] 1000+ 文章列表滚动流畅
- [ ] 关键词列表虚拟化正常
- [ ] 返回列表时恢复滚动位置

---

#### 2.4 数据分页加载

**需求描述**：
- IndexedDB 查询分页
- 滚动加载更多
- 显示加载状态

**实现步骤**：
1. 扩展 db api 支持分页查询
2. 实现 `useInfiniteScroll` Hook
3. 改造文章列表加载逻辑

**验收标准**：
- [ ] 首次加载只加载前 20 条
- [ ] 滚动到底部自动加载更多
- [ ] 加载过程显示 loading 指示器

## 🛠️ 开发规范

### 每个任务的开发流程

1. **创建任务分支**：`git checkout -b feature/xxx`
2. **实现功能**：按照实现步骤编码
3. **自测**：对照验收标准逐项检查
4. **代码检查**：运行 `npm run lint`
5. **类型检查**：运行 `npx tsc --noEmit`
6. **合并**：功能完成后合并到主分支

### 代码规范

- 使用 TypeScript 严格类型
- 组件使用函数式组件 + Hooks
- 新增功能需添加注释说明
- 遵循现有代码风格和命名规范
